# Developer Guide

Conventions, architecture, and workflow for contributors.

## Project layout

```
2do-aggregator
├── app/                    // aggregator engine (never shipped to standalone deploy)
│   ├── Helpers/
│   ├── Models/
│   ├── Services/
│   └── Shared/             // code shared between app and standalone bundle
├── assets/                 // static resources (images, fonts) — never auto-generated
│   ├── fonts/
│   └── images/
├── bin/                    // compiled artefacts and maintenance scripts
│   ├── aggregator          // compiled PHAR (generated)
│   ├── cron.sh
│   └── deploy.sh
├── bootstrap.php           // included by all modules
├── bundle/                 // generated by dev/build.php — never edit directly
│   └── standalone/         // standalone mini-site bundle
├── cache/
├── config/
├── data/                   // generated by aggregator (cron) — never edit directly
├── dev/                    // developer tools (build, smoke-test, start-server)
├── lib/                    // external libraries
├── logs/
├── public/                 // reserved for future Laravel app — do not use yet
├── src/                    // THE source to edit
│   └── bundle/
│       ├── lsl-board/      // git submodule — 2do-board LSL script
│       └── standalone/     // standalone bundle sources
├── tests/
└── vendor/
```

## Placement rules

> *"Does this file need to exist on a standalone deploy server?"*

| Answer | Location |
|--------|----------|
| Yes — runtime code for the standalone site | `src/bundle/standalone/` |
| Shared between app and standalone | `app/Shared/` |
| App-internal (aggregator, helpers, models) | `app/` |
| External libraries | `lib/` |
| CLI entry points and maintenance scripts | `bin/` |

**`bundle/` is 100% generated** by `dev/build.php`. Never edit anything inside it.
**`public/`** is reserved for the future Laravel app.
**`assets/`** holds static resources (images, fonts). Never auto-generated.

---

## Three-step workflow

1. **build** (manual, during development) — `dev/build.php`
   Compiles `src/bundle/standalone/` → `bundle/standalone/index.php` (PHAR)
   and copies static assets. Run after any source change.

2. **refresh** (cron + on demand) — `bin/aggregator`
   Fetches upstream calendars → `data/`. Never touches `bundle/`.

3. **deploy** (cron + on demand) — `bin/deploy.sh`
   rsyncs `bundle/standalone/` and `data/` to configured targets.
   No transformation — pure transport.

Cron runs **refresh + deploy** only. Build is a developer action.

## Git conventions

### Branches
- Work on `dev`. `master` is stable and public.
- Only the project owner merges `dev → master` and pushes.

### Commit message format

```
(scope) short subject

- detail
- detail

Optional additional context.
```

- **scope**: area of the change — e.g. `(api)`, `(lsl)`, `(build)`, `(tests)`, `(doc)`, `(config)`
- **subject**: imperative, lowercase, no trailing period
- **details**: bullet list with `-`, one item per logical change
- Omit details for trivial single-change commits
- Prefix with `(untested)` when the change has not been verified yet; reword after a successful test

### Rules
- Never push. Pushing is the project owner's responsibility.
- Never sign commits, never add co-authorship lines.
- Never skip hooks (`--no-verify`).

## Testing

```bash
./tests/run-tests.sh
```

Requirements tests (`tests/Requirements/`) need a running dev server.
Start it with `dev/start-server.sh` before running the full suite.

Unit tests and feature tests run without a server.

See `tests/README.md` for details.

## Code style

- PHP 8.2+, checked by `phpcs` with PHPCompatibility.
- English for all code, comments, and documentation.
- Prefer simple, well-tested constructs. Avoid global mutable state.
- Reuse existing libraries and patterns already in the project.
- Small, focused unit tests for new logic.
